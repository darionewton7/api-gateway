Arquitetura

Diagrama de Arquitetura

Componentes da Arquitetura

VPC & Networking

Rede isolada com subnets públicas e privadas em múltiplas zonas de disponibilidade, garantindo alta disponibilidade e segurança.

ECS/Fargate

Serviço gerenciado para execução de containers, eliminando a necessidade de gerenciar servidores.

RDS PostgreSQL

Banco de dados relacional gerenciado com backups automáticos e alta disponibilidade.

Lambda & API Gateway

Processamento serverless de pagamentos com exposição via API RESTful.

Módulos Terraform

VPC

Configura a rede virtual com subnets públicas e privadas, Internet Gateway, NAT Gateway, tabelas de rotas e NACLs.

terraform 
module "vpc" {
  source = "../../modules/vpc"
  
  project     = local.project
  environment = local.environment
  
  vpc_cidr        = "10.0.0.0/16"
  azs             = ["us-east-1a", "us-east-1b"]
  private_subnets = ["10.0.1.0/24", "10.0.2.0/24"]
  public_subnets  = ["10.0.101.0/24", "10.0.102.0/24"]
}
Security

Define grupos de segurança para controle de tráfego entre os componentes da aplicação.

terraform 
module "security" {
  source = "../../modules/security"
  
  project     = local.project
  environment = local.environment
  
  vpc_id   = module.vpc.vpc_id
  vpc_cidr = module.vpc.vpc_cidr_block
}
ECS

Configura o cluster ECS/Fargate para execução de containers, incluindo IAM roles e CloudWatch logs.

terraform 
module "ecs" {
  source = "../../modules/ecs"
  
  project     = local.project
  environment = local.environment
  
  vpc_id               = module.vpc.vpc_id
  private_subnets      = module.vpc.private_subnets
  ecs_security_group_id = module.security.ecs_security_group_id
}
RDS

Configura o banco de dados PostgreSQL gerenciado, incluindo subnet groups, parameter groups e backups.

terraform 
module "rds" {

Gateway de Pagamento com Stripe

Função Lambda

Função serverless que processa pagamentos via API do Stripe.

javascript 
const AWS = require('aws-sdk');
const secretsManager = new AWS.SecretsManager();
const stripe = require('stripe');

exports.handler = async (event) => {
  try {
    // Parse do corpo da requisição
    const body = JSON.parse(event.body);
    
    // Validação dos dados de pagamento
    if (!body.cardNumber || !body.expMonth || 
        !body.expYear || !body.cvc || !body.amount) {
      return formatResponse(400, { 
        error: 'Dados de pagamento incompletos' 
      });
    }
    
    // Processa o pagamento
    const result = await processPayment(body);
    
    if (result.success) {
      return formatResponse(200, result);
    } else {
      return formatResponse(400, result);
    }
  } catch (error) {
    console.error('Erro ao processar a requisição:', error);
    return formatResponse(500, { 
      error: 'Erro interno do servidor' 
    });
  }
};
Segurança

Implementação segura com AWS Secrets Manager e IAM roles com permissões mínimas.

terraform 
resource "aws_secretsmanager_secret" "stripe" {
  name        = "${local.name}/stripe-api-key"
  description = "Chave de API do Stripe para o gateway de pagamento"
  
  tags = local.tags
}

resource "aws_secretsmanager_secret_version" "stripe" {
  secret_id     = aws_secretsmanager_secret.stripe.id
  secret_string = jsonencode({
    api_key = var.stripe_api_key
  })
}

resource "aws_lambda_function" "payment_processor" {
  function_name = "${local.name}-payment-processor"
  
  vpc_config {
    subnet_ids         = var.private_subnets
    security_group_ids = [var.lambda_security_group_id]
  }
  
  environment {
    variables = {
      STRIPE_SECRET_KEY_ARN = aws_secretsmanager_secret.stripe.arn
      ENVIRONMENT           = var.environment
    }
  }
}
API Gateway

Endpoint HTTP para integração com o gateway de pagamento.

terraform 
resource "aws_apigatewayv2_api" "payment_api" {
  name          = "${local.name}-payment-api"
  protocol_type = "HTTP"
  
  cors_configuration {
    allow_origins = ["*"]
    allow_methods = ["POST", "OPTIONS"]
    allow_headers = ["Content-Type", "Authorization"]
    max_age       = 300
  }
}

resource "aws_apigatewayv2_route" "payment_processor" {
  api_id    = aws_apigatewayv2_api.payment_api.id
  route_key = "POST /process-payment"
  target    = "integrations/${aws_apigatewayv2_integration.payment_processor.id}"
}

Pipeline CI/CD com GitHub Actions

Workflow Principal

Pipeline completo para testes, validação e deploy.

yaml 
name: CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

jobs:
  test:
    name: Testes
    runs-on: ubuntu-latest
    steps:
      - name: Checkout código
        uses: actions/checkout@v3

      - name: Configurar Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Executar testes
        run: cd payment-gateway && npm test

  terraform-validate:
    name: Validar Terraform
    runs-on: ubuntu-latest
    needs: test
    steps:
      - name: Checkout código
        uses: actions/checkout@v3

      - name: Configurar Terraform
        uses: hashicorp/setup-terraform@v2

      - name: Validar módulos Terraform
        run: |
          cd terraform/modules/vpc && terraform validate
Segurança

Workflow para verificação de segurança e vulnerabilidades.

yaml 
name: Security Scan

on:
  schedule:
    - cron: '0 0 * * 0'  # Executa todo domingo à meia-noite
  workflow_dispatch:

jobs:
  security-scan:
    name: Verificação de Segurança
    runs-on: ubuntu-latest
    steps:
      - name: Checkout código
        uses: actions/checkout@v3

      - name: Verificar dependências Node.js
        run: |
          cd payment-gateway
          npm audit --production

      - name: Verificar segurança do Terraform
        uses: aquasecurity/tfsec-action@v1.0.0
        with:
          working_directory: terraform
Detecção de Drift

Workflow para detectar diferenças entre a infraestrutura atual e o estado desejado.

yaml 
name: Infrastructure Drift Detection

on:
  schedule:
    - cron: '0 6 * * 1-5'  # Executa de segunda a sexta às 6h
  workflow_dispatch:

jobs:
  drift-detection:
    name: Detecção de Drift da Infraestrutura
    runs-on: ubuntu-latest
    strategy:
      matrix:
        environment: [staging, production]
    
    steps:
      - name: Verificar Drift
        id: plan
        run: |
          cd terraform/environments/${{ matrix.environment }}
          terraform plan -detailed-exitcode -out=tfplan
          echo "EXITCODE=$?" >> $GITHUB_ENV
Ambientes

Configuração de ambientes separados para staging e produção.

Desenvolvimento

Trabalho em branches de feature com testes locais.

Pull Request

Testes automatizados e validação do Terraform.

Staging

Deploy automático após merge para branch develop.

Produção

Deploy automático após merge para branch main.

Pipeline CI/CD

Fluxo de Trabalho

1. Desenvolvimento

Desenvolva novas features em branches de feature e faça commit das mudanças.

2. Pull Request

Abra um Pull Request para a branch develop. O pipeline CI/CD executará automaticamente os testes e validações.

3. Deploy em Staging

Após aprovação, faça merge do PR para develop para deploy automático em staging.

4. Testes em Staging

Teste as mudanças no ambiente de staging.

5. Deploy em Produção

Abra um PR de develop para main. Após aprovação, faça merge para deploy automático em produção.

Benefícios do Pipeline CI/CD

  • Automação Completa

    Testes, validação e deploy automatizados, reduzindo erros humanos.

  • Feedback Rápido

    Detecção imediata de problemas durante o desenvolvimento.

  • Segurança Integrada

    Verificações de segurança automatizadas em cada etapa.

  • Ambientes Consistentes

    Garantia de que staging e produção são configurados de forma idêntica.

Guia de Início Rápido

Siga estes passos para começar a usar o projeto:

  1. Clone o repositório:
    bash 
    git clone https://github.com/seu-usuario/devops-project.git
cd devops-project
  2. Configure os secrets no GitHub conforme documentado em .github/SECRETS.md.
  3. Inicialize o estado remoto do Terraform:
    bash 
    cd terraform/environments/staging
terraform init
  4. Aplique a infraestrutura:
    bash 
    terraform apply

Considerações de Segurança

Práticas de segurança implementadas no projeto:

  • Rotação de Credenciais

    Implemente rotação regular de credenciais da AWS e do Stripe.

  • Atualizações de Segurança

    Mantenha as dependências atualizadas usando o workflow de security scan.

  • Privilégio Mínimo

    Todas as IAM roles seguem o princípio do privilégio mínimo.

  • Criptografia

    Dados em repouso e em trânsito são criptografados.
